# Skills

Project-specific skills will live under `skills/<name>/SKILL.md`.

`M001` does not add project-specific skills yet. The directory is kept stable so later milestones can add narrow skills without bloating `AGENTS.md` or `CLAUDE.md`.

When skills are added, each one should declare:

- when to trigger
- when not to trigger
- which files to read first
- its output contract
