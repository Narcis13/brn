# T01: Write structured continuation and checkpoint artifacts

## Objective

Persist machine-readable continuation packets and deterministic recovery checkpoints for every canonical run.

## Why Now

Recovery cannot be reproducible if Phase 3 only leaves behind markdown continuation notes.

## Acceptance Criteria

- Every next-action dispatch writes `continuation.json` beside `continue.md`.
- Every run writes a pre-dispatch checkpoint and a post-result checkpoint under `checkpoints/`.
- Existing `current.json.recovery_ref` remains compatible with the new artifacts.
