# T01: Add deterministic recovery assessment and reconcile commands

## Objective

Decide recovery from disk state alone and expose it through `recover show` and `recover reconcile`.

## Why Now

Phase 6 is not real if interrupted runs still depend on hidden operator judgment or naive resume heuristics.

## Acceptance Criteria

- Recovery assessment explains whether a run should resume, dispatch fresh, retry, replan, or await human input.
- `recover show` returns the current assessment for an explicit or implicit target run.
- `recover reconcile` persists the assessment, writes a reconcile checkpoint, and updates deterministic phase/state without dispatching work.
